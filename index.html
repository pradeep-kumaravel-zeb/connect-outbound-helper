📖 How to Use This Console

1. Overview

This console lets you preview all connect calls that the Lead Nurturing cron job would make for a given franchise, and then manually trigger them one at a time. No calls are fired automatically — you control each one.

2. Configuration Fields

FieldDescription
API Base URLThe base URL of the Bot Builder backend (e.g., UAT or production). The preview and call endpoints are appended to this.
System IDThe system ID to run the preview for. This determines which system's franchises and data are used.
Franchise IDThe specific franchise to preview. Only this franchise's leads/customers will be processed.

3. Running a Preview

  • Fill in the API Base URL, System ID, and Franchise ID.
  • Click "Run Preview".
  • The system calls POST /schedule/v1/lead-nurturing-preview which runs the full nurturing logic in dry-run mode — no emails are sent and no calls are made.
  • The response includes all pending connect calls and email counts.
The preview runs the exact same logic as the real cron job, but with dry_run=true. This means it respects all business hours, permissions, schedules, and circuit breakers.

4. Understanding the Results

After a successful preview, you'll see:

  • Summary bar — Total connect calls, Track 1 emails, and Track 2 emails.
  • Connect calls table — Each row is a pending outbound call.
ColumnDescription
#Row number.
PhoneThe phone number that will be called.
Lead/Customer IDThe entity type and ID (e.g., "Lead 12345").
FranchiseThe franchise ID this call belongs to.
PurposeWhy this call is being made. Color-coded badges:
🔴 Pending Job — Has approved proposals pending scheduling.
🟠 Unattended Quote — Has open quotes with no action taken.
🟣 Rejected Quote — Has retired/rejected quotes for re-engagement.
🔵 Lead Follow-up — New lead follow-up (Track 2).
PriorityCall priority level (1 = highest).
MessageThe call script/message. Hover to see the full text.
ActionClick "Call" to trigger this specific call.

5. Triggering a Call

  • Click the green "Call" button on any row.
  • This sends a POST /schedule/v1/lead-nurturing-call with that specific call's payload.
  • The button disables while the request is in flight.
  • ✓ Initiated (green) — The call was successfully queued.
  • ✗ Failed (red) — Something went wrong. The button re-enables so you can retry.
Each click triggers a real outbound call. This is not a simulation. Only click "Call" when you intend to actually dial the customer/lead.

6. Typical Workflow

  • Step 1: Set the API URL to your UAT/staging environment.
  • Step 2: Enter the System ID and Franchise ID you want to test.
  • Step 3: Click "Run Preview" to see what calls would be made.
  • Step 4: Review the table — check phone numbers, purposes, and messages.
  • Step 5: Click "Call" on individual rows to trigger them one by one.

7. Troubleshooting

IssueResolution
Preview returns 0 callsCheck that the franchise has active LCN configuration, is within business hours, and has leads/quotes in the validation window.
"Request failed" errorVerify the API Base URL is correct and the server is running. Check browser console for CORS errors.
Call shows "✗ Failed"The connect endpoint rejected the call. Check server logs for details (e.g., invalid phone number, connect service down).
No "Lead Follow-up" callsTrack 2 connect calls require the lead to have allow_calls=true and the franchise must have connect follow-up enabled.
